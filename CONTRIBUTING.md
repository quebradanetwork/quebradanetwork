# Contribuindo com a Quebrada Network

Ficamos felizes com o seu interesse em contribuir! Siga as diretrizes abaixo para garantir que o processo seja ágil e organizado.

## Como Contribuir

1. **Faça um Fork** do repositório.
2. **Crie uma Branch** para sua feature ou correção (`git checkout -b feature/minha-feature`).
3. **Faça os Commits** de suas alterações com mensagens claras e descritivas.
4. **Envie para o seu Fork** (`git push origin feature/minha-feature`).
5. **Abra um Pull Request** detalhando o que foi alterado e o motivo.

## Padrões de Código

* Mantenha o código limpo e organizado.
* Siga a indentação padrão do projeto.
* Documente funções complexas.
* Realize testes básicos antes de abrir o Pull Request.

## Reportando Bugs

Se você encontrou um bug, por favor, use a aba **Issues** do repositório e selecione o template de "Reportar Bug". Forneça o máximo de detalhes possível, incluindo como reproduzir o erro.
